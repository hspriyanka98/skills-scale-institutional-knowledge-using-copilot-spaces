# OctoAcme Project Management Documentation

## Welcome
This documentation hub contains all the process guides, templates, and best practices used by OctoAcme to run successful projects. Use this as your central resource for understanding how we approach project delivery.

## Project Management Overview
OctoAcme uses a **customer-first, iterative approach** to project delivery with clear ownership, data-informed decisions, and psychological safety. Our methodology emphasizes:

- **Customer value and usability**: Prioritize what delivers real value to users
- **Small, testable increments**: Ship iteratively to gather feedback early
- **Named accountability**: Each project has a Project Manager & Product Lead with clear responsibilities
- **Evidence-based iteration**: Measure impact and adjust based on data
- **Collaborative feedback culture**: Foster psychological safety and encourage learning from all team members

### Core Principles
- **Scope**: Applies to all cross-functional projects that deliver product features, services, or integrations
- **Roles**: Clear definition of Project Manager, Product Manager, Developers, QA/Testing, and Stakeholders
- **Communication**: Weekly PM + PdM syncs, twice-weekly team standups, monthly stakeholder updates, and ad-hoc escalations
